Перед вами небольшая статья-шпаргалка,  которая включает в себя команды удобные, но не всем знакомые команды Git, которые могут стать вашими постоянными спутниками в разработке.

1. git init: Клонирование проекта как репозитория Git.

2. git clone: Клонирование репозитория в новую директорию.

3. git config: Настройка значений конфигурации Git.

4. git remote add: Подключение к удаленному репозиторию.

5. git remote -v: Просмотр подключенных удаленных репозиториев.

6. git status: Просмотр статуса файлов в локальном репозитории.

7. git add: Добавление файлов для коммита.

8. git reset: Отмена изменений и сброс состояния проекта.

9. git commit: Фиксация изменений с сообщением.

10. git push: Отправка закоммиченных файлов в удаленный репозиторий.

11. git fetch: Получение последней версии локального репозитория.

12. git pull: Извлечение и загрузка информации из удаленного репозитория.

13. git branch: Создание, просмотр, переименование и удаление веток.

14. git checkout: Переключение на другие ветки.

15. git merge: Объединение двух веток.

16. git merge -a: Объединение веток с автоматическим выбором изменений.

17. git merge — abort: Эта команда используется для отмены слияния.

18. git diff: Эта команда используется для сравнения изменений.

19. git rebase: Склеивает коммиты, переписывая историю.

20. git stash: Сохраняет неподтвержденные изменения в отдельном хранилище.

21. git stash pop: Применяет ранее отложенные изменения.
